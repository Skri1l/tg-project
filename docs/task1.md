Цель задания
В этом семестре мы будем разрабатывать интеллектуальный агрегатор уведомлений LinkTracker — Telegram-бота, который отслеживает изменения на веб-страницах и оперативно информирует пользователя.

Проект будет реализован поэтапно в рамках трёх сервисов: Bot (взаимодействие с пользователем через Telegram API), Scrapper (мониторинг контента и отправка обновлений) и AI Agent (интеллектуальная обработка данных).

Мы начнём с простой архитектуры — два сервиса и REST — а к концу курса построим продвинутую систему с брокером сообщений, где искусственный интеллект будет фильтровать, группировать и суммировать события, превращая поток уведомлений в персональную сводку.

В этом домашнем задании вы разработаете базовый каркас Telegram-бота, реализовав обработку команд /start и /help. Он станет основой для дальнейшего расширения функциональности в последующих заданиях.

В результате выполнения задания у вас должен получиться рабочий прототип бота, готовый к интеграции с внешними API. Для неподдерживаемых команд можно предусмотреть базовую реакцию (например, ответ с уведомлением о том, что команда не поддерживается).

Обратите внимание: не нужно реализовывать весь функционал будущего проекта. Достаточно выполнить только те требования, которые указаны в этом задании.

Функциональные требования
Бот должен поддерживать следующие команды:
/start - начало работы пользователя. При первом запуске может приветствовать пользователя (например: "Добро пожаловать! Используйте /help, чтобы посмотреть доступные команды.").
/help - вывод списка доступных команд.
На неизвестные команды бот должен отвечать сообщением об ошибке, например: "Неизвестная команда. Воспользуйтесь /help, чтобы посмотреть список доступных команд."
Нефункциональные требования
Токен авторизации должен храниться в конфигурационном файле, недоступном для общего доступа
Используйте только те библиотеки, которые уже включены в предоставленный шаблон проекта. Подключение дополнительных зависимостей запрещено без предварительного согласования в чате курса.
Используйте типобезопасную конфигурацию. Пример для Java, Python, Go, Scala, IOptions<T> в .NET
Используйте структурное логирование (добавление key-value значений к логу, вместо зашивания данных в поле message) в коде приложения. Подумайте об этом в конце, когда основной код будет написан
Бонусные требования
Бонус: реализуйте автоматическую настройку меню команд в интерфейсе Telegram. При запуске бот должен вызывать метод setMyCommands (см. Telegram Bot API ), чтобы отобразить команды /start и /help в списке команд бота.

Технологии и примеры
Для каждого языка рекомендуется использовать официальные или популярные библиотеки для работы с Telegram Bot API (например, python-telegram-bot, telegram-bot-api для Go и т.д.), они уже включены в шаблон.

Пошаговый план реализации
Советы по реализации
Изучите общее описание проекта, чтобы было понятно, какое из приложений за что отвечает: у вас есть 2 приложения, которые общаются между собой по HTTP. В данном домашнем задании Вы должны реализовать один сервис — каркас телеграм-бота.
SDK для работы с API предоставляет низкоуровневый набор методов. Почти всегда над SDK приходится делать "обёртку" для комфортной работы. Изучите SDK и подумайте, как лучше всего её реализовать.
В проекте можно создавать новые модули, пакеты или неймспейсы, если вам нужно переиспользовать код между несколькими приложениями.
Разбор входного сообщения и диспетчеризация по типам команд - достаточно стандартная задача. Попробуйте спроектировать архитектуру своего приложения так, чтобы команды можно было легко добавлять и удалять.
Что не надо делать в этом домашнем задании?
Работы с БД и системами очередей будут реализованы в дальнейших домашних заданиях, а пока можно считать, что всё хранится в памяти в контейнерах типа "Словарь" или "Список". Для этого реализуйте паттерн Repository — это упростит замену на работу с БД в будущем.
Цель этого задания — разобраться с механизмом работы с Telegram API и реализовать диспетчеризацию команд. Функционал управления ссылками (добавление, валидация, получение списка) будет реализован во втором домашнем задании.
